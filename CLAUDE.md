# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Real-time voice assistant web app. Users speak into their microphone, see a live transcript, and hear AI responses read aloud. Static SPA hosted on Cloudflare Pages with all backend logic in Convex cloud.

**Stack:** React 18 + Vite 6 + Tailwind CSS 3 | Convex backend | Claude Haiku 4.5 via `@convex-dev/agent` + `@ai-sdk/anthropic` | Browser-native Web Speech API (STT/TTS)

## Commands

```bash
npm run dev              # Convex sync + Vite dev server (concurrent)
npm run dev:frontend     # Vite only (port 5173)
npm run dev:convex       # Convex sync only

npm run build            # Production build → dist/
npm run typecheck        # TypeScript strict check (frontend + convex)
npm run lint             # ESLint, zero-warning policy (--max-warnings 0)
npm run lint:fix         # ESLint auto-fix
npm run format           # Prettier write
npm run format:check     # Prettier check

npm run test             # Vitest watch mode
npm run test:run         # Single Vitest run
npm run test:coverage    # Coverage report
npm run test:e2e         # Playwright all browsers
npm run test:e2e:ui      # Playwright UI mode
npm run ci               # Full CI: typecheck → lint → test → build
```

Run a single test file: `npx vitest run path/to/test.ts`

## Architecture

```
Browser (Cloudflare Pages SPA)
  └─ React App
       ├─ Web Speech API (mic → text, text → speech)
       └─ Convex Client (WebSocket)
              └─ Convex Cloud
                   ├─ sessions table
                   ├─ @convex-dev/agent (threads + messages)
                   └─ Actions → Claude Haiku 4.5
```

**UI state machine:** `idle → listening → processing → speaking → idle` — defined as `AppStatus` type in `src/App.tsx`. This is the single mental model for the entire UI.

**Data flow:** VoiceButton captures speech → calls `sendMessage` action → action invokes agent → agent writes response to DB → reactive query updates Transcript → TTS speaks response.

## Convex Patterns

- **Mutations** are synchronous DB writes. **Actions** call external APIs (LLM). Actions write back via `ctx.runMutation`.
- **Queries** are reactive — `useQuery`/`usePaginatedQuery` auto-update the frontend.
- `ctx.scheduler.runAfter(0, ...)` triggers async work (thread init) after session creation.
- Use `internal` for server-only functions (scheduler, inter-function). Use `api` for frontend-callable functions.
- **Never edit `convex/_generated/`** — auto-generated by `npx convex dev`.
- Backend env vars are set in the Convex dashboard, not in `.env` files. Only `VITE_CONVEX_URL` goes in `.env.local`.

## Testing

- **Backend tests** (`convex/__tests__/`): `convex-test` + Vitest in `edge-runtime` environment.
- **Frontend tests** (`src/__tests__/`): `@testing-library/react` + Vitest in `jsdom` environment.
- **E2E** (`e2e/`): Playwright. Voice features need manual testing (browser API limits).
- Environment separation in `vitest.config.ts` via `environmentMatchGlobs`: `convex/**` → edge-runtime, `src/**` → jsdom.

## Code Style

- TypeScript strict mode with `noUnusedLocals`/`noUnusedParameters`.
- ESLint 9 flat config. Zero warnings enforced.
- PascalCase for React components, camelCase for Convex functions.
- `@/` alias maps to `src/`. Convex imports use relative paths within `convex/`.
- Unused function args prefixed with `_` (e.g., `_ctx`).
- No default exports except where required by framework.

## Deployment

```bash
npx convex deploy --cmd 'npm run build'   # Backend + frontend build
npx wrangler pages deploy dist --project-name convex-voice-demo  # Cloudflare
```

CI/CD: push to `main` → GitHub Actions → typecheck → lint → test → deploy (Convex + Cloudflare Pages). PRs run checks only.
